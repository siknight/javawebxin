1.计算机的体系结构(熟悉)
1.1 什么是计算机？
  计算机俗称“电脑”,是一种用于高级算计并被广泛使用的设备。
  计算机主要由两大部分组成：计算机硬件 +  计算机软件。

1.2 常见的硬件
  在计算机中常见的硬件：CPU、内存、硬盘、主板、显示器、鼠标、键盘等等。
     CPU - 中央处理器，是计算机核心的部件，类似于人的大脑。
         - 所有的运算通常都在CPU进行。 
     内存 - 是计算机的存储部件，用于存储数据内容的。
          - 该存储部件中存放的数据并不是永久存储的，不保存关机会丢失数据。
          - 该存储部件中的数据内容是可以直接被CPU访问的，效率比较高。
     硬盘 - 是计算机的存储部件，用于存储数据内容的。
          - 该存储部件中存储的数据是永久保存的，前提就是硬盘没有损坏。
          - 该存储部件中的数据内容是不可以被CPU直接访问的，需要加载到内存中。

科普：
   1Tb = 1024Gb
   1Gb = 1024Mb
   1Mb = 1024Kb
   1Kb = 1024byte(字节)   
   1byte = 8bit(二进制位) 
   计算机的底层只认识由0和1组成的二进制序列。

1.3 常见的软件
   计算机的软件主要分为两大类：系统软件 + 应用软件。
   其中系统软件就是指操作系统，应用软件就是指安装在操作系统上的软件。
   目前主流的操作系统：Windows系列/Linux系列/Unix系列
       Windows系列：Xp、Vista、Win7、Win8、Win8.1、Win10。
   目前主流的应用软件：微信、QQ、迅雷、Office等

1.4 计算机的体系结构
   使用者 => 应用软件 => 系统软件 => 硬件部件
        系统软件分为：系统外壳(Shell) + 系统内核(Kernel)。

2.java语言的概述(熟悉)
2.1 基本概念
  java语言本质上就是一种用于实现人机交互的语言机制。
  java语言诞生于1995年，刚开始隶属于sun公司，现在隶属于oracle-sun公司(甲骨文)，在编程语言排行榜上占据重要的地位。  

2.2 主要版本
(1)Java SE版本   
   Java Platform Standard Edition - Java平台标准版
      - 主要用于编写类似于Office那样的桌面软件。
(2)Java EE版本
   Java Platform Enterprise Edition - Java平台企业版
      - 主要用于编写电信计费系统、电子商务网站等具有B/S架构的项目。
(3)Java ME版本
   Java Platform Mirco Edition - Java平台微型版
      - 随着Android系统的普及，已经走向淘汰。

3.开发环境的搭建(重点)
3.1 下载和安装jdk
(1)下载方式
    方式一：从www.oracle.com/www.sun.com官网上进行下载。
    方式二：从百度/谷歌/搜狗等网站搜索jdk1.7版本下载和安装。

(2)安装方式
    若下载的是安装版，双击之后一路点击下一步，切记不要有中文路径名。
    若下载的是绿色版，只需要解压即可。            
    
3.2 相关的概念
    jdk - java开发工具包，只要编写和测试java代码就必须安装jdk，jdk中包含jre。
    jre - java运行时环境，只要运行java代码就必须安装jre。
    javac.exe - java编译器，用于将java代码进行编译翻译成字节码文件。
    java.exe - java解释器，用于使用字节码代码进行边解释边执行。
    jvm - java虚拟机，并不是一个真实存在的主机，用于实现跨平台的效果。
    ide - 集成开发环境，用于高效编写java程序的工具。

3.3 环境变量的配置
(1)基本概念
   环境变量就是指用于存放软件环境路径信息的变量，变量就是指数值可以改变的量。
   通常情况下，使用可执行文件需要加上路径才可以，当把该可执行文件的路径加入到环境变量Path之后，就可以省略路径直接通过可执行文件名来启动该软件。
   
(2)配置方式
   计算机 => 右键，选择属性 => 高级系统设置 => 高级 => 系统变量 => 找到Path后，点击编辑 => 将javac.exe/java.exe所在的路径信息放到变量值的最前面，使用分号
隔开 => 一路点击确定即可。
   切记不要将Path变量值原来的内容删除，否则后果很严重！！！
   配置完毕之后重新打开dos窗口进行测试，在dos窗口中输入 javac 回车，若没有出现javac不是内部或外部的命令....则表示配置成功。

3.4 编写java程序的流程
   (1)新建一个记事本，将该文件的名称命名为xxx.java。
   (2)编写代码并进行保存。
   (3)启动dos窗口，切换到上述.java文件所在的目录。
   (4)使用javac xxx.java 进行编译生成字节码文件.class。
   (5)使用java xxx 进行解释并执行字节码文件。
